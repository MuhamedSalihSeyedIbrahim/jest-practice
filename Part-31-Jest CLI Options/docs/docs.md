<header class="postHeader"><a class="edit-page-link button" href="https://github.com/facebook/jest/edit/master/docs/CLI.md" target="_blank" rel="noreferrer noopener">Edit</a><h1 id="__docusaurus" class="postHeaderTitle">Jest CLI Options</h1></header><article><div><span><p>The <code>jest</code> command line runner has a number of useful options. You can run <code>jest --help</code> to view all available options. Many of the options shown below can also be used together to run tests exactly the way you want. Every one of Jest&apos;s <a href="/docs/en/configuration">Configuration</a> options can also be specified through the CLI.</p>
<p>Here is a brief overview:</p>
<h2><a class="anchor" aria-hidden="true" id="running-from-the-command-line"></a><a href="#running-from-the-command-line" aria-hidden="true" class="hash-link"><svg class="hash-link-icon" aria-hidden="true" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"/></svg></a>Running from the command line</h2>
<p>Run all tests (default):</p>
<pre><code class="hljs css language-bash">jest
</code></pre>
<p>Run only the tests that were specified with a pattern or filename:</p>
<pre><code class="hljs css language-bash">jest my-test <span class="hljs-comment">#or</span>
jest path/to/my-test.js
</code></pre>
<p>Run tests related to changed files based on hg/git (uncommitted files):</p>
<pre><code class="hljs css language-bash">jest -o
</code></pre>
<p>Run tests related to <code>path/to/fileA.js</code> and <code>path/to/fileB.js</code>:</p>
<pre><code class="hljs css language-bash">jest --findRelatedTests path/to/fileA.js path/to/fileB.js
</code></pre>
<p>Run tests that match this spec name (match against the name in <code>describe</code> or <code>test</code>, basically).</p>
<pre><code class="hljs css language-bash">jest -t name-of-spec
</code></pre>
<p>Run watch mode:</p>
<pre><code class="hljs css language-bash">jest --watch <span class="hljs-comment">#runs jest -o by default</span>
jest --watchAll <span class="hljs-comment">#runs all tests</span>
</code></pre>
<p>Watch mode also enables to specify the name or path to a file to focus on a specific set of tests.</p>
<h2><a class="anchor" aria-hidden="true" id="using-with-yarn"></a><a href="#using-with-yarn" aria-hidden="true" class="hash-link"><svg class="hash-link-icon" aria-hidden="true" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"/></svg></a>Using with yarn</h2>
<p>If you run Jest via <code>yarn test</code>, you can pass the command line arguments directly as Jest arguments.</p>
<p>Instead of:</p>
<pre><code class="hljs css language-bash">jest -u -t=<span class="hljs-string">&quot;ColorPicker&quot;</span>
</code></pre>
<p>you can use:</p>
<pre><code class="hljs css language-bash">yarn <span class="hljs-built_in">test</span> -u -t=<span class="hljs-string">&quot;ColorPicker&quot;</span>
</code></pre>
<h2><a class="anchor" aria-hidden="true" id="using-with-npm-scripts"></a><a href="#using-with-npm-scripts" aria-hidden="true" class="hash-link"><svg class="hash-link-icon" aria-hidden="true" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"/></svg></a>Using with npm scripts</h2>
<p>If you run Jest via <code>npm test</code>, you can still use the command line arguments by inserting a <code>--</code> between <code>npm test</code> and the Jest arguments.</p>
<p>Instead of:</p>
<pre><code class="hljs css language-bash">jest -u -t=<span class="hljs-string">&quot;ColorPicker&quot;</span>
</code></pre>
<p>you can use:</p>
<pre><code class="hljs css language-bash">npm <span class="hljs-built_in">test</span> -- -u -t=<span class="hljs-string">&quot;ColorPicker&quot;</span>
</code></pre>
<h2><a class="anchor" aria-hidden="true" id="camelcase--dashed-args-support"></a><a href="#camelcase--dashed-args-support" aria-hidden="true" class="hash-link"><svg class="hash-link-icon" aria-hidden="true" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"/></svg></a>Camelcase &amp; dashed args support</h2>
<p>Jest supports both camelcase and dashed arg formats. The following examples will have equal result:</p>
<pre><code class="hljs css language-bash">jest --collect-coverage
jest --collectCoverage
</code></pre>
<p>Arguments can also be mixed:</p>
<pre><code class="hljs css language-bash">jest --update-snapshot --detectOpenHandles
</code></pre>

<h2><a class="anchor" aria-hidden="true" id="reference"></a><a href="#reference" aria-hidden="true" class="hash-link"><svg class="hash-link-icon" aria-hidden="true" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"/></svg></a>Reference</h2>
<h3><a class="anchor" aria-hidden="true" id="jest-regexfortestfiles"></a><a href="#jest-regexfortestfiles" aria-hidden="true" class="hash-link"><svg class="hash-link-icon" aria-hidden="true" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"/></svg></a><code>jest &lt;regexForTestFiles&gt;</code></h3>
<p>When you run <code>jest</code> with an argument, that argument is treated as a regular expression to match against files in your project. It is possible to run test suites by providing a pattern. Only the files that the pattern matches will be picked up and executed. Depending on your terminal, you may need to quote this argument: <code>jest &quot;my.*(complex)?pattern&quot;</code>. On Windows, you will need to use <code>/</code> as a path separator or escape <code>\</code> as <code>\\</code>.</p>
<h3><a class="anchor" aria-hidden="true" id="--bail"></a><a href="#--bail" aria-hidden="true" class="hash-link"><svg class="hash-link-icon" aria-hidden="true" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"/></svg></a><code>--bail</code></h3>
<p>Alias: <code>-b</code>. Exit the test suite immediately upon <code>n</code> number of failing test suite. Defaults to <code>1</code>.</p>
<h3><a class="anchor" aria-hidden="true" id="--cache"></a><a href="#--cache" aria-hidden="true" class="hash-link"><svg class="hash-link-icon" aria-hidden="true" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"/></svg></a><code>--cache</code></h3>
<p>Whether to use the cache. Defaults to true. Disable the cache using <code>--no-cache</code>. <em>Note: the cache should only be disabled if you are experiencing caching related problems. On average, disabling the cache makes Jest at least two times slower.</em></p>
<p>If you want to inspect the cache, use <code>--showConfig</code> and look at the <code>cacheDirectory</code> value. If you need to clear the cache, use <code>--clearCache</code>.</p>
<h3><a class="anchor" aria-hidden="true" id="--changedfileswithancestor"></a><a href="#--changedfileswithancestor" aria-hidden="true" class="hash-link"><svg class="hash-link-icon" aria-hidden="true" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"/></svg></a><code>--changedFilesWithAncestor</code></h3>
<p>Runs tests related to the current changes and the changes made in the last commit. Behaves similarly to <code>--onlyChanged</code>.</p>
<h3><a class="anchor" aria-hidden="true" id="--changedsince"></a><a href="#--changedsince" aria-hidden="true" class="hash-link"><svg class="hash-link-icon" aria-hidden="true" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"/></svg></a><code>--changedSince</code></h3>
<p>Runs tests related to the changes since the provided branch. If the current branch has diverged from the given branch, then only changes made locally will be tested. Behaves similarly to <code>--onlyChanged</code>.</p>
<h3><a class="anchor" aria-hidden="true" id="--ci"></a><a href="#--ci" aria-hidden="true" class="hash-link"><svg class="hash-link-icon" aria-hidden="true" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"/></svg></a><code>--ci</code></h3>
<p>When this option is provided, Jest will assume it is running in a CI environment. This changes the behavior when a new snapshot is encountered. Instead of the regular behavior of storing a new snapshot automatically, it will fail the test and require Jest to be run with <code>--updateSnapshot</code>.</p>
<h3><a class="anchor" aria-hidden="true" id="--clearcache"></a><a href="#--clearcache" aria-hidden="true" class="hash-link"><svg class="hash-link-icon" aria-hidden="true" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"/></svg></a><code>--clearCache</code></h3>
<p>Deletes the Jest cache directory and then exits without running tests. Will delete <code>cacheDirectory</code> if the option is passed, or Jest&apos;s default cache directory. The default cache directory can be found by calling <code>jest --showConfig</code>. <em>Note: clearing the cache will reduce performance.</em></p>
